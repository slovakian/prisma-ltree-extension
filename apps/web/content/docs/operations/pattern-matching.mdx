# Pattern Matching Operators

Pattern matching operators filter `ltree` paths using PostgreSQL `lquery` and `ltxtquery` syntax. Pattern arguments are **strings** (or `string[]` for array matching) passed through `param()` ŌĆö they are not separate column types.

```typescript
import { db } from "../prisma/db";
import { param } from "@prisma-next/sql-query/param";

const category = db.schema.tables.category;
```

## matchesLquery()

Match a path against a single `lquery` pattern.

```typescript
const plan = db.sql
  .from(category)
  .select({ id: category.columns.id, path: category.columns.path })
  .where(category.columns.path.matchesLquery(param("pattern")))
  .build({ params: { pattern: "*.computers.*" } });

await db.runtime().execute(plan);
```

Common `lquery` syntax:

- `*` ŌĆö matches exactly one label
- `*{n}` / `*{n,m}` ŌĆö quantifiers on wildcard labels
- `{a,b,c}` ŌĆö matches any one of the labels at that position
- `|` ŌĆö alternation within a label

**SQL equivalent:** `path ~ $1::lquery`

### Examples

**Paths exactly two labels deep:**

```typescript
db.sql
  .from(category)
  .where(category.columns.path.matchesLquery(param("pat")))
  .build({ params: { pat: "*.*" } });
```

**Paths under `electronics` at any depth:**

```typescript
db.sql
  .from(category)
  .where(category.columns.path.matchesLquery(param("pat")))
  .build({ params: { pat: "electronics.*" } });
```

**Direct children of `electronics` only:**

```typescript
db.sql
  .from(category)
  .where(category.columns.path.matchesLquery(param("pat")))
  .build({ params: { pat: "electronics.*{1}" } });
```

## matchesLqueryArray()

Match if the path satisfies **any** pattern in a `string[]`.

```typescript
const plan = db.sql
  .from(category)
  .select({ id: category.columns.id, path: category.columns.path })
  .where(category.columns.path.matchesLqueryArray(param("patterns")))
  .build({
    params: { patterns: ["electronics.*", "software.*"] },
  });

await db.runtime().execute(plan);
```

**SQL equivalent:** `path ? $1::lquery[]`

## matchesLtxtquery()

Match using `ltxtquery` full-text-style patterns over path labels (words combined with `&`, `|`, `!`).

```typescript
const plan = db.sql
  .from(category)
  .select({ id: category.columns.id, path: category.columns.path })
  .where(category.columns.path.matchesLtxtquery(param("query")))
  .build({ params: { query: "computer | phone" } });

await db.runtime().execute(plan);
```

**SQL equivalent:** `path @ $1::ltxtquery`

## Pattern syntax reference

| Syntax               | Meaning            | Example                                            |
| -------------------- | ------------------ | -------------------------------------------------- |
| `*`                  | One label          | `a.*.c` matches `a.b.c`                            |
| `*` with quantifiers | Bounded wildcards  | `Top.Science.*{1}` ŌĆö one label under `Top.Science` |
| `{a,b}`              | Label alternatives | `{a,b}.c` matches `a.c` or `b.c`                   |
| `\|` (ltxtquery)     | Boolean OR         | `computer \| phone`                                |
| `&` (ltxtquery)      | Boolean AND        | `computer & laptop`                                |

Refer to [PostgreSQL ltree documentation](https://www.postgresql.org/docs/current/ltree.html) for full pattern syntax.

## ORM lane (optional)

await db.orm.Category.where((c) => c.path.matchesLquery("electronics.*"))

# prisma-ltree

PostgreSQL `ltree` brings hierarchical tree data structures to **Prisma Next**. The extension pack provides typed codecs, query operators, and a baseline migration for querying tree-structured data without raw SQL.

## What is ltree?

The PostgreSQL `ltree` type represents a label tree — a path-like data type designed for hierarchical queries. Trees are stored as dot-separated labels (e.g. `Top.Science.Astronomy` or `electronics.computers.laptops`).

Common use cases:

- Organization hierarchies
- Filesystem-like structures
- Category trees
- Taxonomies with ancestor/descendant queries

# prisma-ltree

PostgreSQL `ltree` brings hierarchical tree data structures to Prisma. The extension provides operators and functions for querying tree-structured data efficiently.

## What is ltree?

The PostgreSQL `ltree` type represents a label tree — a path-like data type designed for hierarchical queries. Trees are stored as dot-separated labels (e.g., `1.2.3` or `root.child.grandchild`).

Common use cases:

- Organization hierarchies
- Filesystem-like structures
- Category trees
- Organizational trees with ancestor/descendant queries
